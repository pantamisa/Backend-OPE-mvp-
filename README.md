Hacer arquitectura C4


En esta sección deben presentar los cuatro niveles del modelo C4: 
 
Nivel 1. Contexto 
Nivel 2. Contenedores 
Nivel 3. Componentes 
Nivel 4. Código / Clases (dominio y patrones) 
 
Deben incluir en cada nivel: 
Imagen del diagrama 
Enlace editable 
Explicación breve 
 
Recuerden: la arquitectura no debe aparecer desconectada del trabajo previo. Debe guardar relación con los requisitos, historias de usuario, MVP y diagrama de clases definidos en el Entregable 1. 


Importante 
Deben analizar y justificar cinco (5) patrones de diseño. 
Los patrones no se pueden repetir. 
Cada patrón debe aplicarse a una necesidad concreta del sistema dentro del alcance del MVP. 
La selección debe evitar sobreingeniería. 
La implementación debe ser viable en tiempo, complejidad y coherencia con el proyecto. 
 
Obligatorio: Deben diligenciar y anexar la plantilla de análisis y selección de patrones GoF. Esta plantilla debe completarse para justificar la pertinencia de cada patrón, su relación con el MVP, las alternativas consideradas, su aplicación concreta al diseño y la validación de que los patrones seleccionados no se repiten ni se solapan de manera innecesaria.
